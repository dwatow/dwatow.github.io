再一次存到上次的commit

¶再一次存到上次的commit

原文連結: https://darkblack01.blogspot.com/2013/12/commit.html
移植時的最後更新日期: 2014-05-27T11:18:09.339+08:00

認錯[1]的情況
反悔上一次的commit
完全重新覆蓋掉上一次的紀錄

都適用。

將目前最新的進度存到上一次的commit上，而且可以修改commit內容。
git commit --amend

然後，如果你上次的commit已經push了，然後local又改掉最後一次的commit要再次的push會失敗。

這時就要使用強制覆
git push --fource
