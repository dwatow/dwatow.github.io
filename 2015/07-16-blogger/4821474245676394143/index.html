應用行動科技紀錄與研究人們日常生活行為與脈絡 7/16 HPX72 from Armuro Chang

這是今天的活動主題。主要是在講傳統的研究方法，和藉由Mobile的研究方法，有什麼差異、優缺點大剖析。

正好最近在看HCI的線上課程。是幾年前史丹佛的Scott開的線上課程。也有提到「研究方法」。

我的筆記如下

傳統的方法

  • Survey
  • Qualitative Method
  • Diary Study

問題設計，要注意要使用者做的是
Retrospective(回顧) v.s. Introspective(內省)

傳統方法 v.s. Real Time Data Caption

為何回顧不Reliable？
因為記憶會記不得呀~
生活的一部份，無須記得
這種無意識的事，就算記，也記不準
  1. Encoding Problem
  2. Storage Problem
  3. Retrieval Failure
  4. Reconstruction Error

Survey和訪談，有些時候很難達到目的，
或者，受試者面對不想回答的問題

Memory Cues幫助回憶？

透過Mobile的持性，可以做到

Real time data caption

現代常用的方法

Experience Sampling Method(ESM)

也稱為Ecological Momentary Assessment(EMA)

早期的ESM工具，就是BB Call+手寫，PDA+手寫，手機+url

優點:

  • 沒有Recall Error的問題
  • 資料貼近現實生活的行為和情境
  • 可研究較大量的受試者

缺點:

  • 你需要肯配合的受試者
  • 測試時可能中斷受試者的活動(煩)
  • 整個測試不宜太長
  • 不適合
    • 需要注意力的活動(測試本質成為生活的干擾)
    • 太頻繁或太少發生的行為
    • 問題的回答，需要打很多字的…申論

Sensing & Logging

加上Eco-Feedback

優點:

  • 大量觀察資料, 可找趨勢, 相關性和行為預測。
  • Log有雜訊，但是誠實
  • Scalable, 且有現成的App

缺點:

  • 秏電, 隱私、安全性顧慮
  • 可觀察趨勢…呃…但是要自己補腦
  • 需要操作和分析資料的人材(工程師)


行動科技研究近年趨勢

Diary+Logging

優

  • 針對特定事件，問受試者問題。(補腦)
  • 受試者回應用來註釋該期間收集的資料
  • 紀錄不同形態的資料，幫助受試著回憶
  • 受試者回應可以解釋紀錄，紀錄可以驗證受試者回應

缺

  • Diary=秏電
  • 自己開發工具(需要工程師)
  • 整合不同形態的Data給受試者檢視(需要工程師)

Wearable Life-logging(穿戴式相機)
Wearable logging+Diary

優

  • 準確度高
  • 資料比Sensor易讀, 情境資料豐富
  • 和手機的Logging 一起使用
  • 放在Diary幫助受試者回憶

缺

  • 受試著及其朋友的隱私，必須拿下來
  • 需要大量的人力的整理與分析
  • 多數照片，沒有資訊性
  • 拍攝速度、容量、畫質好的好設備，少！
    (電力>1天, 容量>2天)
